## Why You Need a Deep Link

Regular links open web pages. That works fine on desktop, but on mobile it creates unnecessary friction.

When someone on their phone clicks a link to a YouTube video and gets sent to a browser that then asks them to open the app, half of them give up. A deep link skips that step entirely. It opens the YouTube app directly, on the right video, in one tap.

That is the difference between a regular link and a deep link, and it matters a lot for engagement, conversions, and user experience.

The good news is that creating one takes about 30 seconds and costs nothing.

Want to understand the underlying concept first? Read [what is a deep link and how does it work](/blog/what-is-a-deep-link).

## What You Need Before You Start

Just one thing: the URL of the content you want to link to.

That is it. No account. No API key. No SDK. No code. Just a URL.

For most platforms, a regular web URL is enough. For example:

- A YouTube video URL like `https://youtube.com/watch?v=...`
- An Instagram profile like `https://instagram.com/username`
- A WhatsApp link like `https://wa.me/1234567890`
- A Google Maps location
- A TikTok profile or video
- Any custom URI scheme

If you have that, you are ready.

Paste your destination URL into the input field. The tool immediately scans the URL and detects the platform.

For example, if you paste `https://www.youtube.com/@channelname`, it detects YouTube and configures the deep link to open the YouTube app at that channel.

Once the platform is detected, you will see a confirmation showing:

Click the "Compose Link" button. Your smart deep link is ready in under a second.

## How to Test Your Deep Link

Always test before you share.

**On iOS:** Tap the link from the Safari browser or Messages app on a real iPhone. Do not use the iOS Simulator for this test — it does not have the same app-launch behavior as a real device.

**On Android:** Tap the link from Chrome on a real Android phone.

**On desktop:** Open the link in a browser and check that the web fallback works correctly.

Check three things in each test:

1. Does it open the right app?
2. Does it land on the right content inside the app?
3. If the app is not installed, does the fallback work?

## Common Issues and How to Fix Them

**The link opens a browser instead of the app.** This usually means the app is not installed on the test device. Install the app and test again. If the app is installed and it still opens a browser, check that you pasted the full URL correctly.

**The link opens the app but not the right screen.** Some platforms have restrictions on which content can be deep linked. For example, private Instagram accounts and some region-restricted YouTube videos may open the app's home screen instead of the specific content.

**The link does not work inside Facebook or Instagram's in-app browser.** Facebook and Instagram deliberately block some app-to-app redirects inside their in-app browsers as an anti-spam measure. Test your links outside these browsers to confirm they work correctly on the open web.

**Custom URI schemes are not routing correctly.** Make sure your URI scheme is registered in your app's `Info.plist` (iOS) or `AndroidManifest.xml` (Android). DeepLinkOS handles the routing, but the app itself needs to be configured to handle incoming URI links.

## When to Use Deep Links vs. Deferred Deep Links

The links you create with DeepLinkOS are standard smart links. They work perfectly for:

- Existing users who already have the app installed
- Content sharing and social campaigns
- Email CTAs and SMS campaigns
- Bio links and QR codes
- Any situation where you want to open a specific app screen

If you need to route new users to a specific in-app screen even after a fresh install (for onboarding flows or paid campaigns), that is deferred deep linking. It requires a tool like Branch.io or AppsFlyer. Read the [Firebase Dynamic Links alternatives guide](/blog/firebase-dynamic-links-alternative) to understand which tools support deferred deep linking and which are free.

## Creating Multiple Links at Scale

If you need to create many deep links, for a large product catalog or a multi-platform campaign, the same process applies for each URL. Work through your list systematically:

1. Group your URLs by platform
2. Create the deep link for each
3. Organize them with descriptive slugs so they are easy to track
4. Test a sample from each group before rolling out the full campaign
